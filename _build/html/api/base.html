Base object types in RepLAB¶

The following classes are at the base of many RepLAB objects.

  • Str: provides infrastructure for pretty printing of objects, in a way compatible with Matlab and Octave

  • Domain: describes a set of elements that can be compared for equality, and can also be sampled from

Str¶

class replab.Str¶

Bases: handle

Defines a ‘str’ default method and overloads ‘disp’

Also provides methods additionalFields and hiddenFields to guide long form object pretty printing

Domain¶

class replab.Domain¶

Bases: replab.Str

Describes a set of elements with a common structure

Those elements can be compared (eqv), and random elements can be produced (sample).

Class members list

  • additionalFields – Returns the name/value pairs corresponding to additional fields to be printed

  • assertEqv – Compares two elements for equality

  • assertNotEqv – Compares two elements for inequality

  • disp –

  • eq – Equality test

  • eqv – Tests domain elements for equality/equivalence

  • headerStr – Tiny single line description of the current object type

  • hiddenFields – Returns the names of the fields that are not printed as a row vector

  • lambda – Constructs a domain from function handles

  • longStr – Multi-line description of the current object

  • sample – Samples an element from this domain

  • shortStr – Single line text description of the current object
